# R2 WebDAV Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an optional Cloudflare R2 WebDAV feature with API-token permission gating, bucket setup, WebDAV credentials, and web UI file management.

**Architecture:** Add a focused `internal/r2dav` service that owns R2 settings, permission availability, bucket operations, an R2-backed `afero.Fs`, WebDAV auth, and file API operations. Keep server handlers thin and extend the existing feature toggle contract additively. The frontend adds an R2 tab and feature row with explicit setup guidance and recoverable error messages.

**Tech Stack:** Go 1.26, `cloudflare-go`, `github.com/fclairamb/afero-s3`, `github.com/spf13/afero`, `golang.org/x/net/webdav`, `bcrypt`, vanilla HTML/CSS/JS.
